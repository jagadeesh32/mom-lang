# Contributor Code of Conduct

mom is a working community of practitioners. The rules below are short,
plain-language, and enforced.

## Standards

- Be precise. Disagreements about code, design, or process belong in
  technical arguments, not personal ones.
- Assume competence. Most surprising behaviour is the result of
  context you don't have, not malice.
- Credit work. If you finish someone else's idea, name them.
- Keep public spaces focused on the project. Off-topic conversations
  belong elsewhere.

## Unacceptable behaviour

- Harassment of any contributor in any project space — issues, PRs,
  discussions, chat, conferences, social media that links to the
  project — for any reason.
- Deliberately misrepresenting another contributor's position or work.
- Publishing a contributor's private information (location, employer,
  personal contact details) without consent.
- Using project channels for commercial spam.

## Reporting

Send concerns to **conduct@mom-lang.dev**. Reports are read by the
maintainers listed in `MAINTAINERS.md` (excluding anyone named in the
report). You will get an acknowledgement within 2 business days.

## Enforcement

The maintainers may:

1. Issue a private warning.
2. Issue a public warning in the affected channel.
3. Suspend the contributor from project spaces for a defined period.
4. Permanently revoke commit / discussion access.

The action taken is proportional to the harm caused and the contributor's
willingness to course-correct. Decisions can be appealed in writing to
the same address; appeals are read by maintainers not involved in the
original decision.

## Scope

This code applies in every project space and at every project event,
online or in person.
