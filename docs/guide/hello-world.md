# Hello World Program

The simplest Mom program. This page walks you through every step from writing the file to running the native binary.

---

## Step 1 — Write the Program

Create a file named `hello.mom`:

```mom
fn main():
    print("Hello, world!")
```

That's the entire program. Let's understand what each part means:

| Part | Meaning |
|---|---|
| `fn main()` | Declares a function named `main`. The colon `:` begins the function body. |
| `    print(...)` | Calls the built-in `print` function. Indentation (4 spaces or 1 tab) defines the block. |
| `"Hello, world!"` | A string literal. |

---

## Step 2 — Run It (Interpreter)

The fastest way — no compile step:

```bash
mom run hello.mom
```

Output:

```
Hello, world!
```

---

## Step 3 — Compile to a Native Binary

```bash
mom build hello.mom -o hello
./hello
```

Output:

```
Hello, world!
```

The binary is statically linked and has no runtime dependency beyond the C standard library.

## A More Interesting Hello World

Let's add a name parameter and a greeting function:

```mom
fn greet(name: String) -> String:
    "Hello, " + name + "!"

fn main():
    print(greet("Mom"))
    print(greet("world"))
```

Run it:

```bash
mom run greet.mom
```

```
Hello, Mom!
Hello, world!
```

---

## Hello World with a Loop

```mom
fn main():
    let names = ["Alice", "Bob", "Carol"]
    for name in names:
        print("Hello, " + name + "!")
```

```
Hello, Alice!
Hello, Bob!
Hello, Carol!
```

## Hello World with Error Handling

```mom
fn try_greet(name: String) -> Result[String, String]:
    if len(name) == 0:
        Err("name cannot be empty")
    else:
        Ok("Hello, " + name + "!")

fn main():
    match try_greet("Mom"):
        Ok(msg)  => print(msg)
        Err(e)   => print("Error: " + e)

    match try_greet(""):
        Ok(msg)  => print(msg)
        Err(e)   => print("Error: " + e)
```

```
Hello, Mom!
Error: name cannot be empty
```

## Project Structure for Larger Programs

When your program grows, create a project:

```bash
mom new my_project
cd my_project
```

This creates:

```
my_project/
├── mom.toml       # project manifest
├── src/
│   └── main.mom   # entry point
└── tests/
    └── main_test.mom
```

```bash
mom run             # run src/main.mom
mom build           # build to ./target/my_project
mom test            # run all tests
```
